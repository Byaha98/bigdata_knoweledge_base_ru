# 5

Ранее я уже делал попытку сделать ознакомительный задачник для pyspark

Здесь же автор как для малышей раскладывает основы Spark.

Единственное, мне кажется, слишком большой упор в идет RDD, но это даже полезно, когда раскладываешь знания по полочкам и смотришь как оно там под капотом. 

В современном спарке есть более высокоуровневые API (например, Dataframe), которые могут работать без постоянного обращения к RDD, используют подкапотные оптимизаторы, что полезно для аналитиков. Тем не менее, насколько я понимаю, хорошо написанные руками аппки  в rdd api будут весьма быстрыми по исполнению, но требуют лучшего понимания спарка.
